# 一般利用者テスト実施手順

## 対象
まず3〜5記事で実施します。

- CTR改善が目的の記事
- 順位改善が目的の記事
- 本文を十分に渡した記事
- 本文不足で確認要求になる記事
- 内部リンク候補がある記事

## 1記事ごとの流れ
1. 改善依頼JSONを作る
2. Claudeの新しいチャットへ貼る
3. 回答が日本語で、指定JSON構造になっているか確認する
4. 元記事にない事実、URL、数値が作られていないか確認する
5. タイトル、導入、見出し、FAQ、内部リンク候補を確認する
6. 結果JSONを保存する
7. 評価テンプレートへ記録する

## 合格の目安
- 重大な事実誤認がない
- 日本語が自然で、そのまま修正案として使える
- 元記事の良い部分を壊していない
- 内部リンクURLを創作していない
- 情報不足時に推測せず確認を求める
- 初心者が手順だけで再現できる

重大な問題が1件でもあれば公開テストを停止し、記事ID・入力・出力・問題点を記録してください。

- 本文不足時は `manual_review_required` を確認する
