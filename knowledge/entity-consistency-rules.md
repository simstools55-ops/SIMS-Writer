# Entity Consistency Rules v1.0

## 1. エンティティ台帳

記事監査時に、次を一時的に台帳化する。

```json
{
  "entities": {
    "products": [],
    "companies": [],
    "people": [],
    "services": [],
    "laws_and_systems": [],
    "dates_and_years": [],
    "prices": [],
    "versions": [],
    "measurements": []
  }
}
```

## 2. 正規化

比較用に次を正規化する。

- 全角・半角
- 大文字・小文字
- ハイフン
- 空白
- 通貨記号
- 桁区切り
- 略称
- 税表記

正規化は検出用であり、正式表記を勝手に変更する根拠にはしない。

## 3. 商品・サービス名

公式表記が確認できる場合は公式表記を優先する。

同一記事で、似た別商品と混同しない。

例：

```text
LPLP ヘアカラートリートメント
LPLP エッセンスカラートリートメント
```

シリーズ名が似ていても別商品として扱う。

## 4. 人名・肩書き

- 人名の漢字
- 敬称
- 所属
- 肩書き
- 発言の帰属

を確認する。

肩書きは時期により変わるため、Evidence Audit対象にも引き継ぐ。

## 5. 年度

タイトルに年号がある場合、本文の主要情報がその年に対応しているかを確認する。

年号だけ差し替える更新を禁止する。

## 6. 価格

次を分けて保持する。

- 通常価格
- セール価格
- 初回価格
- 月額
- 年額
- 税込
- 税別
- 送料込み
- 本体のみ

## 7. バージョン

OS、アプリ、製品世代、制度改定版を区別する。

例：

```text
Windows 11 23H2
Windows 11 24H2
```

単に「Windows 11」で統一すると条件差を消す場合がある。

## 8. 数値範囲

「約」「最大」「平均」「目安」などの修飾語を数値と一体で扱う。

```text
約200円
最大200円
平均200円
```

は同一値ではない。

## 9. 不明な場合

正しい値を推測して統一しない。

次のいずれかとする。

- warningを付ける
- 条件を明示する
- 断定を弱める
- 人間確認を求める
